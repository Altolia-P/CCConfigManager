## 三栏宽度

| 栏 | 宽度 | 说明 |
|----|------|------|
| 左侧（导航） | 200px | 固定宽度，类型切换 |
| 中间（列表） | 350px | 固定宽度，可滚动 |
| 右侧（详情） | flex | 填充剩余空间 |

## 左侧导航栏

### 结构
```
═══ 配置类型 ═══
● Skills      107↗ 19↘
○ Agents       49
○ Commands     53↗ 28↘
○ Rules        10↗ 40↘
─── 来源筛选 ───
☑ ECC
☑ Gstack
☑ Superpowers
☑ 独立
☐ 未知
─── 状态筛选 ───
☑ 活跃
☑ 隔离
```

### 行为
- 点击类型 → 切换列表内容，取消其他类型高亮
- 活跃/隔离数字 → 点击可直接筛选
- 来源筛选 → AND 逻辑（同时满足类型和来源和状态）
- 灰色数字 ↗ = 活跃数，↘ = 隔离数

## 中间列表

### 每个列表项
```
┌─────────────────────────────┐
│ ● investigate      ★ ecc    │
│   系统调试，四阶段方法论...   │
└─────────────────────────────┘
  ● = 活跃  ○ = 隔离
  ★ = 来源标签（带颜色）
```

### 行为
- 点击 → 右侧显示详情
- 当前选中项高亮
- 搜索时实时过滤
- 滚动加载全部（不分页，200 项不多）

## 右侧详情面板

### 默认状态（未选中任何项）
```
点击左侧列表中的项目查看详情
```

### 已选中状态
```
═══ 详情 ═══

名称: investigate
类型: Skill
来源: ECC
状态: 🟢 活跃
路径: C:\Users\Altolia\.claude\skills\investigate

═══ 描述 ═══
系统调试技能，四阶段方法论：调查、分析、假设、实施。

═══ 内容预览 ═══
---
name: investigate
description: ...
---
(前 20 行内容)

[操作按钮]
┌──────────────┐  ┌──────────────┐
│ ↗ 移到隔离区  │  │ 📋 复制路径  │
└──────────────┘  └──────────────┘
```

### 搬家按钮行为
1. 点击 → 弹出确认对话框："将 investigate 移到隔离区？"
2. 确认 → POST /api/move → 成功 → 列表刷新，该项消失
3. 失败 → 显示错误提示
4. 按钮文字根据当前状态动态变化：活跃 → "移到隔离区"，隔离 → "移到活跃区"

## 搜索栏

- 位置：顶部，横跨全宽
- 占位符：`🔍 搜索...`
- 行为：输入即搜，300ms 防抖
- 搜索范围：名称 + 描述
- 搜索时列表实时过滤，详情清空

## 统计栏

- 位置：底部
- 内容：`共 N 项 · 活跃 X · 隔离 Y · 未知来源 Z`
- 未知来源 > 0 时数字标黄提醒

## 恢复默认视图

### 状态重置
- 切换类型 → 清除选中项，列表回到顶部
- 点击页面空白 → 不清除选中（保留详情查看）
- 清除搜索 → 恢复当前筛选条件下的完整列表

### 无结果状态
```
┌─────────────────────────────┐
│                             │
│       🔍 没有找到匹配项      │
│     试试其他搜索词或清除筛选  │
│                             │
└─────────────────────────────┘
```

### 加载状态
打开页面时，在列表区显示 "加载中..."，数据返回后渲染。
若 API 请求失败，显示错误提示和"重试"按钮。
